Hexoの設定ファイルの各項目(_config.yml)

Hexo(blog framework)の設定ファイル(_config.yml)をいちいち調べている自分に気づきました。面倒くさいったらありゃしないので、この際ある程度まとめておきます。

Site

  • この項目はhtmlのheadタグ内に書きそうな内容というか、そのまんまですね。

title

ウェブページのタイトル。titleタグと、

root

ブログが設定されている場所までのパスを書きます。つまり、ブログがサブディレクトリに設置されており、上述のurlのドメイン名以下にパスが書かれていれば、/サブディレクトリ名/。ルートディクトリに設置しているなら、/になります。

permalink:

記事のリソースのパスの形式を設定します。デフォルトの:year/:month/:day/:title/なら、ルートから/年/月/日/記事ファイル名/という感じになります。

tag_dir

タグページのディレクトリ名

archive_dir

アーカイブ（全件表示）ページのディレクトリ名

category_dir

カテゴリーページのディレクトリ名

code_dir: downloads/code

なにこれ

permalink_defaults:

なにこれ

Directory

  • ローカル側のディレクトリ設定

source_dir

ジェネレート(htmlファイル等諸々化)する前のmarkdownファイルのディレクトリ名

public_dir

ジェネレートした後のhtmlファイル等諸々が入るディレクトリの名前

Writing

  • hexo newで新規ページのmarkdownファイルを作成したときの設定

new_post_name: :year/:month/:title.md

作成されるmarkdownファイルの名前

default_layout: post

作成時に称されるデフォルトテンプレート

titlecase: false

なにこれ

external_link: true # Open external links in new tab

filename_case: 0

なにこれ

render_drafts: false

なにこれ

post_asset_folder: true

作成時にファイル名と同じフォルダを作るか。記事に使用する画像etc格納用

relative_link: false

なにこれ

highlight:

  • enable: true
  • line_number: true
  • tab_replace:

なにこれ

default_category
