ApacheとTomcatってなんだし！

気持ちがはずむし～（大嘘）

開発現場の当たり前を知らない新米の僕は「Apache」と「tomcat」の混在に非常に苦しめらました。

だって

  • tomcat

    Apache Tomcatは、Java ServletやJavaServer Pages (JSP) を実行するためのWebコンテナ（サ ーブレットコンテナ、サーブレットエンジン）である。
    Apache Tomcat - ウィキペディア
    ja.wikipedia.org/wiki/Apache_Tomcat

  • apache

    単にApacheとも称されている。 開発は、Apacheソフトウェア財団のApache HTTPサーバプロジェクトで 行われている。 Apache Licenseの下でソースコードが公開および配布されており、代表的なオープンソ ースソフトウェアの一つである。
    Apache HTTP Server - ウィキペディア
    ja.wikipedia.org/wiki/Apache_HTTP_Server

・・・だよ？

WebコンテナとHTTPサーバだから全然違うじゃん・・・というのもありますが、新人にはこのApacheという省略の仕方は卑怯だ。だってtomcatもApacheじゃん！

そして名前もさることながら、tomcatにWebサーバー機能があるという・・・・なんだし！tomcatでjavaアプリケーションを動かしならwebサーバーとして使えばいいじゃん！と思うわけです。

実際使い方もクリソツ。htmlファイルやらなんやらを設置して、localhostで見る・・・ってだけでもうごちゃごちゃになります。（実際は置くものもポートも違うのだけれど・・・）

TomcatとApacheの連携の理由

こんな具体的な疑問が出るだけでも自分の成長を実感することが出来ることは置いておいて、２つをわざわざ連携させて使用する理由を調べました。

TomcatはJavaアプリケーションをサーバー側で動かす「Webアプリケーションサーバー」です。前述のとおり、これにはWebサーバ機能も有しますが、このWebサーバ機能は主に開発とデバッグ用に用意されたものらしく、性能に難があるらしいです。（この性能に関しては本記事では割愛）

そこでApacheの出番。こちらはアクセスしてきたクライアントへ何らかのサービスを提供するためのWebサーバソフトウェア。Webアプリケーションサーバーの必要がない、HTMLや画像などの静的コンテンツはより高速なWebサーバーに処理させることで、動的コンテンツのみをアプリケーションサーバー（Tomcat）へ集中させることが出来ます。

なるほどね〜（・８・）
